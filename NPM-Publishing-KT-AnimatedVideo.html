01 — What is NPM?
Understanding the Registry
📦
Node Package Manager

The default package manager for Node.js — manages dependencies, scripts, and packages in the JavaScript ecosystem.

🌐
Public Registry

registry.npmjs.org — Free, open to everyone. 2M+ packages. No auth needed to install.

🔒
Private Registry

GitHub Packages, Verdaccio, AWS CodeArtifact. Requires authentication — used for internal/proprietary code.

⚙️
CLI Tool

The npm command runs from your terminal: install, publish, update, audit packages.

npm vs npx vs yarn

npm — installs & publishes packages

npx — runs packages without installing

yarn / pnpm — alternative package managers

💡 Key Insight:
npm publish pushes your code to the registry so others can npm install it.
02 — Prerequisites
Before You Start

Verify all of these before running a single npm command.

🟢
node --version
Node.js v18+ — npm runs on Node. Older versions may fail during publish. Install from nodejs.org.
🟢
npm --version
npm v9+ — Bundled with Node. Verify it's recent enough for registry tooling.
👤
npmjs.com
Create npm account — Required for publishing public packages. Sign up at npmjs.com/signup.
🐙
github.com
GitHub account — Required for GitHub Packages (private registry).
🔐
2FA
Enable 2FA on npm — Profile → Account Settings → Security → Two-Factor Auth. Required to publish.
03 — Project Setup
package.json — Your Package Identity
package.json
{
  "name": "@yourscope/my-lib",
  "version": "1.0.0",
  "description": "My awesome library",
  "main": "dist/index.js",
  "files": ["dist/", "README.md"],
  "publishConfig": {
    "access": "public"
  },
  "repository": { "url": "https://github.com/you/repo.git" }
}
name

Use @scope/pkg-name for scoped packages. Must be globally unique on npm.

version

SemVer: MAJOR.MINOR.PATCH. Never republish the same version!

files

Whitelist what gets published. Exclude tests/, src/, .env — keeps tarball small.

publishConfig.access

Required for scoped packages to be public. Without it, defaults to private (paid)!

04 — Public Registry
Publishing to npmjs.com (Public)
1
cd 01-public-package → navigate to your package folder
npm publish uses the current directory. You must be inside the package folder.
2
npm install && npm run build && npm test
Build React components into dist/. All 4 tests must pass before publishing.
3
npm pack --dry-run
Preview exactly which files will be published. Verify no secrets are included!
4
npm login → username + password + OTP (2FA)
Stores credentials in ~/.npmrc. Without login, publish returns 401 Unauthorized.
5
npm publish --access public
Required for scoped (@scope) packages. Verify at npmjs.com/package/YOUR-NAME.
05 — Private Registry
Private Package Publishing Options
🐙 GitHub Packages
1. Scoped name: @your-github-org/my-lib
2. Create PAT with write:packages + repo scope
3. Configure .npmrc:
@myorg:registry=https://npm.pkg.github.com
//npm.pkg.github.com/:_authToken=TOKEN
4. npm publish
✅ Best for GitHub-centric teams
🏠 Verdaccio (Self-Hosted)
1. Start:
npx verdaccio # → localhost:4873
2. Configure .npmrc:
registry=http://localhost:4873/
3. Add user:
npm adduser --registry http://localhost:4873/
4. Publish:
npm publish --registry http://localhost:4873/
✅ Full control, offline-capable
Other Options

☁️ AWS CodeArtifact — AWS-native, IAM-integrated

🦊 GitLab Registry — Built into GitLab pipelines

💎 npm Orgs (paid) — Private scoped on npmjs.com

⚠️ Always use scoped names (@org/name) for private packages to avoid registry conflicts.
06 — GitHub Packages: Step-by-Step
Private Package: Full Walkthrough
B1 — Create Personal Access Token

GitHub → Settings → Developer settings → Tokens (classic). Check read:packages, write:packages, repo. Copy immediately!

B2 — Create GitHub Repository

github.com/new — links the package to your repo. Set visibility (private repo = private package).

B4 — Update package.json
"name": "@johndoe/tutorial-private-utils",
"publishConfig": { "registry": "https://npm.pkg.github.com/johndoe" }
B5 — Create .npmrc
# cp .npmrc.example .npmrc
@johndoe:registry=https://npm.pkg.github.com
//npm.pkg.github.com/:_authToken=ghp_xxx...

⚠️ .npmrc is in .gitignore — never commit tokens!

B6–B8 — Build, Test, Publish
npm install && npm run build && npm test
# 4 passed, 0 failed
npm publish
+ @johndoe/tutorial-private-utils@1.0.0
🔍 Verify at github.com/YOUR-USERNAME?tab=packages
07 — Authentication
.npmrc & Token Management
~/.npmrc — User-level configuration
# Default public registry
registry=https://registry.npmjs.org/
 
# Scope → GitHub Packages
@myorg:registry=https://npm.pkg.github.com
//npm.pkg.github.com/:_authToken=${GITHUB_TOKEN}
 
# Scope → Verdaccio
@internal:registry=http://localhost:4873/
//localhost:4873/:_authToken=${LOCAL_TOKEN}
🚨
NEVER commit tokens to Git! Use environment variables or CI/CD secrets. Add .npmrc to .gitignore if it contains tokens.
Token Types
Read-Only Token
For CI to install private packages. Minimal permissions.
Publish Token
For npm publish in workflows. Requires write:packages.
Automation Token
Bypasses 2FA — safe for GitHub Actions / automated CI.
Granular Token
Per-package / per-org fine-grained access control.
08 — CI/CD Automation
GitHub Actions: Auto-Publish on Tag
.github/workflows/publish.yml
name: Publish Package
on:
  push:
    tags: ['v*']
jobs:
  publish:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
          registry-url: 'https://registry.npmjs.org'
      - run: npm ci
      - run: npm run build
      - run: npm publish --access public
        env:
          NODE_AUTH_TOKEN: $\{{ secrets.NPM_TOKEN }}
Workflow Trigger

Triggered on git tag push matching v*

git tag v1.0.0
git push origin v1.0.0
Setup Steps

1. Generate Automation token at npmjs.com

2. GitHub repo → Settings → Secrets → Actions

3. Add secret NPM_TOKEN

💡 Use Automation token in CI — it bypasses 2FA safely.
09 — Best Practices
Tips, Security & Checklist
🔖
Always Bump Version
Run npm version patch|minor|major before every publish. Same version = error!
👁️
Preview with npm pack
Run npm pack --dry-run to see exactly what files will be published before pushing.
📄
Write a README.md
Shown on npmjs.com. Include install steps, usage, API docs, and examples.
🚫
Use .npmignore / files
Exclude test/, src/, .env, docs/ from the published tarball to keep it lean.
🔐
Enable 2FA
Protect your account at npmjs.com. Use Automation tokens in CI to bypass 2FA safely.
🔍
npm audit Regularly
Scan dependencies for known vulnerabilities before publishing each new version.
10 — Troubleshooting
Common Errors & Fixes
Package name already exists

Choose a different name. Check availability at npmjs.com/package/your-name. Or use a scoped name: @yourusername/pkg-name

403 Forbidden (GitHub Packages)

Token needs write:packages + repo scopes. Package name must start with @YOUR-USERNAME/. Check repository.url in package.json.

401 Unauthorized

Public: Run npm login again. Private: Check .npmrc has correct token; token may have expired.

Cannot publish over existing version

Run npm version patch (or minor / major) before npm publish.

404 Not Found (installing private package)

Consumer project needs its own .npmrc with registry URL and auth token. Package name in npm install must match exactly including scope.
