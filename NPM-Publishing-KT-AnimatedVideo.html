01 — What is NPM?
Understanding the Registry
📦
Node Package Manager

The default package manager for Node.js. Manages dependencies, scripts, and packages in the JavaScript ecosystem.

🌐
Public Registry

registry.npmjs.org
Free, open to everyone. 2M+ packages. No auth needed to install.

🔒
Private Registry

GitHub Packages, Verdaccio, AWS CodeArtifact. Requires authentication. Used for internal/proprietary code.

⚙️
CLI Tool

The npm command-line tool: install, publish, update, audit packages from your terminal.

npm vs npx vs yarn

npm — installs & publishes packages

npx — runs packages without installing

yarn/pnpm — alternative package managers (different lock files)

💡 Key Insight:
npm publish pushes your code to the registry so others can npm install it.

02 — Project Setup
package.json — Your Package Identity
{
  "name": "@yourscope/my-lib",
  "version": "1.0.0",
  "description": "My awesome library",
  "main": "dist/index.js",
  "files": ["dist/", "README.md"],
  "publishConfig": {
    "access": "public"
  }
}
name

Use @scope/pkg-name for scoped packages. Must be globally unique on npm.

version

SemVer: MAJOR.MINOR.PATCH. Never republish the same version!

files

Whitelist what gets published. Exclude tests, src, .env — keeps tarball small.

publishConfig.access

Required for scoped (@scope) packages to be public. Otherwise defaults to private!

03 — Public Registry
Publishing to npm Public Registry
1
npm login
Authenticate with your npm account. Prompts for username, password, OTP.
2
npm pack --dry-run
Preview exactly which files will be published. Verify no secrets are included!
3
npm publish
Publish to the public registry. First publish creates the package on npmjs.com.
4
npm publish --access public
Required for scoped packages (@scope/name). Without this, it defaults to private (paid)!
5
npm version patch && npm publish
Always bump version before re-publishing. patch=bug fix, minor=feature, major=breaking.
04 — Private Registry
Private Package Publishing
🐙 GitHub Packages
1. Name: @your-github-org/my-lib
2. Create PAT with write:packages scope
3. Configure .npmrc:
@myorg:registry=https://npm.pkg.github.com
//npm.pkg.github.com/:_authToken=TOKEN
4. Publish:
npm publish
✅ Best for GitHub-centric teams
🏠 Verdaccio (Self-Hosted)
1. Install & start:
npx verdaccio
# Starts at localhost:4873
2. Configure .npmrc:
registry=http://localhost:4873/
3. Add user:
npm adduser --registry http://localhost:4873/
4. Publish:
npm publish --registry http://localhost:4873/
✅ Full control, offline-capable
Other Options

☁️ AWS CodeArtifact — AWS-native, integrates with IAM

🦊 GitLab Registry — Built into GitLab pipelines

💎 npm Orgs (paid) — Private scoped on npmjs.com

⚠️ Always use scoped package names (@org/name) for private packages to avoid registry conflicts.
05 — Authentication
.npmrc & Token Management
~/.npmrc — User-level configuration
# Default registry
registry=https://registry.npmjs.org/
 
# Scoped → GitHub Packages
@myorg:registry=https://npm.pkg.github.com
//npm.pkg.github.com/:_authToken=${GITHUB_TOKEN}
 
# Scoped → Private Verdaccio
@internal:registry=http://localhost:4873/
//localhost:4873/:_authToken=${LOCAL_TOKEN}
🚨
NEVER commit tokens to Git! Use environment variables or CI/CD secrets. Add .npmrc to .gitignore if it contains tokens.
Token Types
Read-Only Token
For CI to install private packages. Minimal permissions.
Publish Token
For npm publish in workflows. Requires write:packages.
Automation Token
Bypasses 2FA — safe for GitHub Actions / automated CI.
Granular Token
Per-package / per-org fine-grained access control.
06 — CI/CD Automation
GitHub Actions: Auto-Publish on Tag
.github/workflows/publish.yml
name: Publish Package
on:
  push:
    tags: ['v*']
jobs:
  publish:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
          registry-url: 'https://registry.npmjs.org'
      - run: npm ci
      - run: npm run build
      - run: npm publish --access public
        env:
          NODE_AUTH_TOKEN: ${{ secrets.NPM_TOKEN }}
Workflow Trigger

Triggered on git tag push matching v*

git tag v1.0.0
git push origin v1.0.0
Setup Steps

1. Generate token at npmjs.com (Automation type)

2. GitHub repo → Settings → Secrets → Actions

3. Add NPM_TOKEN secret

💡 Use Automation token in CI to bypass 2FA safely.
07 — Best Practices
Tips, Checklist & Security
🔖
Always Bump Version
Run npm version patch|minor|major before every publish. Same version = error!
👁️
Preview with npm pack
Run npm pack --dry-run to see exactly what files will be published before pushing.
📄
Write a README.md
Shown on npmjs.com. Include install steps, usage, API docs, examples.
🚫
Use .npmignore / files
Exclude test/, src/, .env, docs/ from published tarball to keep it lean.
🔐
Enable 2FA
Protect your account at npmjs.com. Use Automation tokens in CI to bypass it safely.
🔍
npm audit Regularly
Scan your dependencies for known vulnerabilities before publishing a new version.
